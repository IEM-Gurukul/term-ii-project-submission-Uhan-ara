***Personal Book Multiverse Tracker***

---

**1. Introduction**

With the increasing popularity of fictional universes such as Harry Potter, Marvel, and anime/manga series, readers often follow multiple interconnected books, characters, and storylines. Managing these fictional worlds and tracking reading progress can become complicated.

The Personal Book Multiverse Tracker is a Java-based console application designed to help users organize books according to universes, series, and characters, while also allowing multiverse connections between universes.

The system allows users to:

-Track books and reading progress

  -Manage fictional universes and series

Record character appearances across books

Search books and characters easily

Explore relationships between connected universes

This project demonstrates core Object-Oriented Programming (OOP) concepts using Java.

2. Objectives

The main objectives of this project are:

To implement Object-Oriented Programming principles in a real-world scenario.

To design a system capable of organizing books into universes and series.

To allow tracking of characters appearing across multiple books and universes.

To provide search functionality for books and characters.

To demonstrate inheritance, abstraction, polymorphism, and encapsulation in Java.

To implement multiverse connections between different fictional worlds.

3. System Features

The system provides the following features:

Universe Management

Add new universes

View all available universes

Connect universes to create multiverse relationships

Series Management

Add series to a universe

View series within a universe

Book Management

Add books to a series

Track reading progress

Display book details

Character Tracking

Add characters to books

Track where characters appear

Allow characters to appear across multiple universes

Search Functionality

Search books by title

Search characters and view their appearances

Default Dataset

The system initializes with predefined universes, series, books, and characters.

File Storage

Save universe data to a text file.

5. OOP Concepts Used

This project demonstrates several fundamental Object-Oriented Programming principles.

5.1 Encapsulation

All class attributes are declared as private, and accessed through getter methods.

Example:

private String title;
private int pagesRead;
5.2 Abstraction

The Book class is declared as an abstract class, defining common properties shared by all book types.

Example:

public abstract class Book
5.3 Inheritance

Specific book types inherit from the abstract Book class.

Example:

public class Novel extends Book
5.4 Polymorphism

Methods such as displayInfo() are overridden in subclasses.

Example:

@Override
public void displayInfo()
5.5 Composition

The system uses hierarchical composition:

Universe
   ↓
Series
   ↓
Book
   ↓
Characters

Each object contains other objects to form a structured system.

7. Class Description
Book (Abstract Class)

Represents a general book with attributes like title, author, and reading progress.

Key Methods:

updateProgress()

displayInfo()

Novel

A subclass of Book representing novels with an additional genre attribute.

Universe

Represents a fictional universe that can contain multiple series and connect with other universes.

Key Methods:

addSeries()

connectUniverse()

Series

Represents a book series containing multiple books.

Uses HashMap to store books.

FicCharacter

Represents fictional characters appearing in books.

Tracks appearances across:

Universe → Series → Book
LibraryManager

Acts as the central controller for managing universes, characters, and searches.

Key Methods:

addUniverse()

searchBook()

searchCharacter()

saveUniverses()

DataInitializer

Loads predefined universes, series, books, and characters when the program starts.
